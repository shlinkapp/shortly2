## Design Context

### Users
Shortly is a public SaaS-style product for external users who need to shorten, share, and manage links quickly with minimal friction. The product also includes richer signed-in workflows for dashboard management, temporary email, API keys, and admin controls, so the interface should help users move from one-off utility to trustworthy account-based usage without feeling overwhelming.

### Brand Personality
Calm, professional, trustworthy. The UI should feel efficient and credible rather than playful or flashy. It should support confidence during link creation and management, especially in admin and dashboard contexts where users are making operational decisions.

### Aesthetic Direction
Light mode first, with a restrained palette and clean visual hierarchy. Current signals in the codebase already point toward a neutral shadcn-based system with Geist typography, subtle borders, low-saturation surfaces, and simple motion. Keep that direction, avoid bright or neon hues, avoid crypto/startup-hype aesthetics, and avoid generic glossy AI-template styling.

### Design Principles
1. Prioritize clarity over novelty — primary actions, status, and link data should read instantly.
2. Build trust through restraint — use neutral color, disciplined spacing, and predictable patterns instead of visual hype.
3. Keep flows lightweight — creation, copying, review, and admin tasks should feel fast and low-friction.
4. Design for operational readability — tables, badges, dialogs, and empty states should support scanning and decision-making.
5. Use motion sparingly — prefer subtle transitions and always avoid effects that add noise or discomfort.
